## API Endpoints


### backend/app/api/v1/admin_auth.py

- **POST** `/login`
  - Authenticate admin user with email and password

    **Features:**
    - Email and password authentication
    - Bcrypt password verification
    - JWT token generation
    - Role-based access (admin, super_admin only)
    - Login tracking

    **Returns:**
    - JWT token for authenticated sessions
    - User profile information

    **Example:**
    ```bash
    curl -X POST "http://localhost:8000/api/v1/admin/auth/login" \\
         -H "Content-Type: application/json" \\
         -d '{
           "email": "nimda.vai@gmail.com",
           "password": "your_password"
         }'
    ```
- **POST** `/verify`
  - Verify JWT token validity

    **Parameters:**
    - token: JWT token to verify

    **Returns:**
    - User information if token is valid
- **GET** `/health`
  - Health check endpoint for admin authentication service

### backend/app/api/v1/admin_products.py

- **GET** `/global`
  - List all products in the global product catalog

    Admin only endpoint for viewing and managing global products
- **GET** `/global/stats`
  - Get statistics about global product catalog
- **GET** `/global/{product_id}`
  - Get statistics about global product catalog

### backend/app/api/v1/analytics.py

- **GET** `/overview`
  - Get comprehensive analytics overview for a store
    
    Returns revenue, orders, customers, and trends data for the specified period.
- **GET** `/products/top`
  - Get top selling products for a store
    
    Returns products ranked by sales volume and revenue for the specified period.
- **GET** `/customers/insights`
  - Get customer behavior insights
    
    Returns customer analytics including new vs returning customers, frequency, and value.
- **GET** `/revenue/trend`
  - Get revenue trend data for charting
    
    Returns daily revenue data for the specified number of days.
- **GET** `/orders/volume`
  - Get order volume trend data for charting
    
    Returns daily order counts for the specified number of days.
- **GET** `/peak-hours`
  - Get peak hours analysis
    
    Returns hourly order distribution to identify peak business hours.

### backend/app/api/v1/auth.py

- **POST** `/send-otp`
  - Send OTP to phone number for development
- **POST** `/login`
  - Simple login for development - accepts any phone with OTP 1234
- **POST** `/verify-otp`
  - Verify OTP - always returns true for OTP 1234 in dev mode
- **POST** `/refresh`
  - Refresh JWT token
- **GET** `/me`
  - Get current user information from token
- **POST** `/send-email-passcode`
  - Send email passcode for authentication
    - Generates a random 6-digit passcode
    - Stores it with 15-minute expiry
    - Single-use only
- **POST** `/verify-email-passcode`
  - Verify email passcode and return auth token
    - Validates passcode exists and hasn't expired
    - Ensures single-use only
    - Limits attempts to 3

### backend/app/api/v1/customer_auth.py

- **POST** `/google`
  - Authenticate customer using Google OAuth
    Verifies Google JWT token and creates/logs in customer
- **POST** `/facebook`
  - Authenticate customer using Facebook OAuth
    Verifies Facebook access token and creates/logs in customer
- **POST** `/register`
  - Register new customer with email and password
- **POST** `/login`
  - Login customer with email and password
- **POST** `/send-otp`
  - Send OTP to phone number
- **POST** `/verify-otp`
  - Verify OTP and login/register customer
- **GET** `/profile`
  - Get customer profile with addresses, payment methods, and order history
    Requires authentication token in Authorization header
- **PUT** `/profile`
  - Update customer profile information
    Requires authentication token in Authorization header
